Design System

The rules this site is built from, extracted from the live pages. Written reference in DESIGN.md; toggle the theme to check both modes.

01 — Color

Warm paper and near-black ink, one blue accent. Every color is a CSS custom property; dark mode swaps the whole set via data-theme. The accent appears only on interactive things: links, hovers, the blockquote rule.

02 — Type

Two faces: Geist for everything read, Geist Mono for everything scanned — dates, labels, chips, captions, chart text. Mono labels are the site’s signature; they are always small, uppercase, letterspaced, and faint.

04 — Charts

Rendered by js/charts.js into .chart containers, styled by css/post.css. Series colors are validated for color-vision safety and contrast in both themes (see DESIGN.md). Chart text always wears text tokens, never a series color. Every chart ships a hover tooltip and a data-table fallback.

05 — Rules
  • The accent is interactive-only: links, hovers, the blockquote rule. Never headings, never body text, never decoration.
  • Mono is metadata-only: labels, dates, chips, captions, chart text, code. Prose never sets in mono.
  • Structure is drawn with 1px hairlines (--line); cards get radius 8–12px and no shadows.
  • Motion is hover feedback and one entry fade — nothing else animates, and prefers-reduced-motion kills all of it.
  • Charts: one y-axis always; never more series than the validated palette (six); assign series colors in slot order, never cycled; direct labels only at line ends and bar tips; grid recessive, data loud.
  • Both themes always: every new color gets a light and a dark value, and charts re-use the shared tokens so the toggle themes them for free.
